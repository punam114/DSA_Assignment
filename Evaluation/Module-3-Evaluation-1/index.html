How the Internet Works?

The Internet works by connecting a vast network of computers and devices through a system of routers and switches, allowing them to communicate and share information. Data is transmitted as packets, following established protocols like IP (Internet Protocol) and TCP (Transport Control Protocol), ensuring reliable and consistent delivery.

Client Server Communication in Operating System

In an Operating System, Client Server Communication refers to the exchange of data and Services among multiple machines or processes. In Client client-server communication System one process or machine acts as a client requesting a service or data, and Another machine or process acts like a server for providing those Services or Data to the client machine. This Communication model is widely used for exchanging data among various computing environments like Distributed Systems, Internet Applications, and Networking Application communication. The communication between Server and Client takes place with different Protocols and mechanisms.

The Journey of a Web Request from browser to server.

A web request journey begins when a user enters a URL into a browser, which then initiates an HTTP request to the server hosting the requested resource. This request travels across the internet, potentially involving DNS resolution, TCP/IP connections, and network hops, before reaching the server. The server processes the request, may interact with databases or other systems, and generates an HTTP response which is then sent back to the browser. Finally, the browser receives and renders the response, displaying the webpage to the user.

  • User Input and DNS Resolution:
    • The user enters a URL (e.g., "https://www.example.com") into their browser.
    • The browser first needs to find the server's IP address. This is done by querying the Domain Name System (DNS).
    • The DNS server resolves the domain name to the corresponding IP address.
  • TCP/IP Connection:
    • The browser establishes a TCP/IP connection with the server's IP address.
    • This connection ensures reliable data transmission between the browser and the server.
  • HTTP Request Transmission:
    • The browser sends an HTTP request to the server, specifying the requested resource (e.g., the HTML page for the URL) and other relevant information.
    • This request includes the request method (e.g., GET, POST), headers, and potentially query parameters.
  • Network Transmission and Routing:
    • The HTTP request data is packaged into packets and transmitted across the network.
    • Routers forward the packets based on the destination IP address, gradually getting closer to the server.
  • Server Processing:
    • The server receives the HTTP request, processes it, and may interact with databases or other systems to generate the requested resource.
    • This processing can involve running server-side scripts, accessing databases, or retrieving data from other sources.
  • HTTP Response:
    • The server generates an HTTP response, which includes a status code (e.g., 200 OK, 404 Not Found), headers, and the content of the requested resource (e.g., the HTML page, JavaScript, CSS, images, etc.).
  • Response Transmission and Rendering:
    • The HTTP response is transmitted back to the browser, following a similar path across the network.
    • The browser receives the response, parses the data (e.g., HTML, CSS, JavaScript), and renders the webpage for the user to view.

Client vs Server

Function Client Server
Initiating Request Sends requests to the server to access resources or services. Responds to client requests, providing data or services.
Processing Data May perform some initial processing of data on the client's device. Handles most of the complex processing and data management.
User Interface Displays the user interface and allows user interaction. Typically does not have a user interface.
Resource Storage Primarily stores data and resources locally, such as cached data. Stores and manages the main data and application logic.
Security May be less secure as it is exposed to the user. Generally more secure, with data and logic protected on the server.
Example A web browser requesting a webpage from a server. A web server serving a webpage to a browser.

Static vs Dynamic Websites

Feature Static Dynamic
